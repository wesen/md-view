## Body (verbatim)

> I want to use [cobra](https://github.com/spf13/cobra) to parse command line arguments.
> The following is a partial rewrite of the code generated by default.
> If you `wails dev`, it will start. However, if I double-click the output executable file from Explorer after doing `wails build`, it does not start. However, it can be started from the command line.
>
> If you run the commented out `run(nil, nil)` in the main function instead of `cmd.Execute()` (without cobra), you can double-click on the built executable to start it.
>
> Also, if I start it from an executable file on the command line, the window is displayed, but it exits immediately from the command line. Is it possible to continue to output stdout and stderr on the command line for post-build debugging?

```go
package main

import (
	"embed"
	"log"

	"github.com/spf13/cobra"

	"github.com/wailsapp/wails/v2"
	"github.com/wailsapp/wails/v2/pkg/logger"
	"github.com/wailsapp/wails/v2/pkg/options"
)

//go:embed frontend/src
var assets embed.FS

//go:embed build/appicon.png
var icon []byte

var cmd = cobra.Command{
	Run: run,
}

func run(_ *cobra.Command, _ []string) {
	// Create an instance of the app structure
	app := NewApp()

	// Create application with options
	err := wails.Run(&options.App{
		Title:      "test",
		Assets:     assets,
		LogLevel:   logger.DEBUG,
		OnStartup:  app.startup,
		OnDomReady: app.domReady,
		OnShutdown: app.shutdown,
		Bind: []interface{}{
			app,
		},
	})

	if err != nil {
		log.Fatal(err)
	}
}

func main() {
	cmd.Execute()
	// run(nil, nil)
}
```

> ### Environment
> Windows 10 Home, Go 1.18, WebView2 99.0.1150.39, npm 8.5.0

## Relevance to MD-WAILS

1. **Cobra + Wails in one binary is confirmed working.** The pattern is: `main()` calls `cobra.Command.Execute()`; a Cobra subcommand's `Run` calls `wails.Run(...)`. This is exactly the shape the drop-in port needs (`md-view view`, `md-view serve`, etc. become Cobra commands; the GUI command calls `wails.Run`).
2. **Known gotcha (2022): double-click launch fails when Cobra requires args.** When launched with no args (double-click), a Cobra root with `Run: run` works, but if the root command is misconfigured it may not start. **Mitigation:** make the root command (no subcommand) default to opening the GUI (e.g. show a file dialog or an empty window), so double-click and bare `md-view` both work. md-view already treats `view` as the primary verb; the port should make bare `md-view` also open the app.
3. **Stdout/stderr on the terminal:** the author notes the CLI returns immediately while the window stays open. This is fine for md-view (the CLI already returns the URL and exits in the current daemon model), but worth knowing: a GUI Wails process does not block the terminal the way a pure CLI does.
4. **This predates `SingleInstanceLock` (v2.7.0).** The drop-in port should combine this Cobra pattern with `SingleInstanceLock` (see `sources/01-wails-single-instance-lock-api.md`) so a second `md-view view b.md` forwards to instance #1.
